## 一、产品定位与核心价值

### 一句话定义

> **面向中文年轻用户的情感梳理 × 关系分析 × 陪伴成长工具。**
> 不是虚拟恋人，不是心理咨询替代品——是帮你看清、想明白、走出来的工具。

### 解决的核心痛点

| 痛点 | 现有解决方案的不足 | 我们的差异化 |
|---|---|---|
| **深夜情绪崩溃，没人可说** | 朋友不在线 / 打扰别人 / 不想被评判 | 24h 即时回应，不评判，先接住情绪再说 |
| **关系中反复纠结，看不清** | 闺蜜立场不中立 / 付费咨询门槛高 | 基于事实的结构化分析，保留不确定性 |
| **不知道怎么跟对方说话** | 自己措辞不当激化矛盾 | 三种语气（温和/直接/幽默）的话术选择 |
| **分手 / 被冷暴力后不知道怎么走出来** | 网上鸡汤没有执行力 | 7天/14天结构化计划 + 每日任务 + 打卡追踪 |
| **情绪反复但看不到自己在好转** | 没有量化工具 | 情绪趋势追踪 + 主动关心 + 成长时间线 |

### 核心哲学（写进代码的产品原则）

1. **共情优先**：先接住情绪，再分析，最后建议。永远不跳过第一步
2. **不替用户做决定**：分析只给框架和视角，"分不分手"是用户自己的事
3. **保留不确定性**：不说"他不爱你"，说"目前的投入模式不对等"
4. **不制造依赖**：禁止"只有我懂你"类表达，鼓励用户回到现实关系中
5. **安全兜底**：高风险用户立即切入安全流程，提供真实求助热线

## 二、六大业务模块详解

### 模块 1：即时情感陪伴（Companion）

**用户场景**：深夜焦虑、被冷暴力后崩溃、反复内耗想找人说说

**核心体验**：
- 用户随便说一段话，系统先用一句话接住情绪（"这种感觉确实很难受"）
- 然后用框架给出洞察（"说实话，从你描述的来看..."）
- 最后给一个当下可执行的小动作（"今晚可以试试..."）
- 以确认式提问结尾（"...是不是？""...知道吧？"）

**底层框架**（源自童锦程情感分析方法论，MIT 协议蒸馏）：
1. **吸引力 > 讨好**：没有人因为你喜欢他而喜欢你
2. **给台阶**：人需要一个说服自己的理由
3. **人性不可考验**：测试对方只会破坏关系
4. **自我炫耀即暴露不安全感**：反复强调什么，就是最担心缺什么
5. **成功前后是两个世界**：人情冷暖是结构性现象，不是针对你

**数据产出**：每轮对话的情绪分类（8 种）+ 问题类型（9 种）+ 风险等级

**当前规模能力**：
- 单轮回复 150-250 字
- 注入最近 6 条历史消息保持上下文连贯
- 注入长期记忆（关系对象 + 事件 + 摘要）实现跨会话连续性
- 语气偏好（温柔/理性/直接）可由用户自选

---

### 模块 2：关系分析（Analysis）

**用户场景**：暧昧期看不清对方意图、分手后想搞明白怎么回事、冷暴力中不确定要不要继续

**核心体验**：
- 用户描述一段情况（自然语言）
- 系统自动抽取客观事实（AI 提取 facts 列表）
- 输出结构化分析卡片：
  - **分析结论**：150-300 字基于事实的整体判断
  - **证据列表**：直接引用用户提到的事实
  - **风险提示**：2-4 条当前关系中值得注意的信号
  - **建议**：一条当下可执行的下一步
  - **置信度**：0-100% 透明展示分析可靠程度

**安全设计**：
- 风险等级 ≥ high 时分析功能自动锁死（切入安全模式）
- 所有结论必须保留不确定性（"看起来..."而非"他就是..."）
- 7 项输出守卫逐条检查，违规自动重生成

**差异化优势**：不是"算命式"判断，是有方法论支撑的结构化推理，用户能看到证据和推理过程

---

### 模块 3：话术教练（Message Coach）

**用户场景**：想给对方发消息但不知道怎么说、怕说错话、想表达需求但担心引起反感

**核心体验**：
- 用户说"帮我写条消息给他"
- 系统输出 3 个版本，各有明确定位：
  - **A 版 · 温和试探**：适合关系敏感期、对方情绪不稳
  - **B 版 · 直接表达**：适合想明确态度、不想再绕弯
  - **C 版 · 轻松幽默**：适合关系基础好、氛围允许玩笑
- 每个版本附 `usage_tip`（适用场景说明），帮用户判断选哪个
- 点击直接复制，不替用户按发送

**设计约束**：
- 每条话术 ≤ 100 字
- 不预测对方反应（"他一定会..."被禁止）
- 不替用户做决定（"你应该发这条"被禁止）

---

### 模块 4：恢复计划（Recovery Plan）

**用户场景**：刚分手想走出来、暧昧期反复纠结想停止内耗

**两种计划**：

| 计划 | 周期 | 适用 | 核心理念 |
|---|---|---|---|
| 7 天走出失恋 | 7 天 | 刚分手、反复联系 | 允许自己悲伤，但每天做一件小事恢复掌控感 |
| 14 天停止内耗 | 14 天 | 暧昧/拉扯/反复纠结 | 识别反刍触发点，建立新的思维习惯 |

**每日任务结构**：
- **任务**：30 分钟内可完成的具体行动（走路、写日记、整理房间等）
- **反思提示**：做完后回答一个问题（"今天最让你在意的是什么？"）
- **鼓励**：一句不鸡汤的认可（"你走到第 3 天了，这本身就不容易"）

**追踪机制**：
- 每日打卡：心情评分（1-10）+ 可选反思文字
- 进度可视化：Day X / Y 进度条
- 历史打卡回顾
- 计划完成后状态归档

---

### 模块 5：记忆与成长追踪（Memory & Growth）

**用户场景**：多次对话后，想看到自己的变化轨迹；不想每次重新解释背景

**三层记忆**：

| 层级 | 内容 | 触发方式 |
|---|---|---|
| **短期记忆** | 当前会话最近 6 条消息 | 每次对话自动注入 |
| **长期记忆** | 关系对象（小A、前任等）+ 关键事件（分手、冷战等）+ 用户画像 | AI 自动抽取，会话结束后异步执行 |
| **会话摘要** | 每次对话的核心议题总结（100-200 字） | 会话结束后 AI 自动生成 |

**成长页**：
- **情绪趋势卡片**：近 7 天情绪指数（1-10）+ 趋势方向（在好转 / 平稳 / 有些低落）
- **关键事件时间线**：按月分组的竖向时间轴（分手、复合、冷战等）
- **关系对象标签**：chips 展示，快速定位"关于谁"
- **会话摘要回顾**：每次对话的精炼总结

**隐私控制**：
- 记忆开关：用户随时关闭 → 系统停止写入任何长期记忆
- 一键删除：彻底清除所有记忆数据，不可恢复
- 高风险内容不入库：risk_level ≥ high 的对话不写入长期记忆

---

### 模块 6：安全与危机处理（Safety）

**用户场景**：用户表达了自杀/自伤意图、极度崩溃、失去现实感

**三层检测**：
1. **关键词匹配**（同步，毫秒级）：正则扫描高危关键词
2. **AI 二次分类**（异步，3 秒内）：理解语境，区分"气死了"和真实危险
3. **输出守卫**（生成后）：确保 AI 回复不含危险内容

**四级响应**：

| 风险等级 | 系统行为 |
|---|---|
| `low` | 正常陪伴或分析 |
| `medium` | 减少刺激性表达，优先陪伴（不进分析模式） |
| `high` | 切入安全模式，禁止分析/话术/恢复计划，提供稳定支持 |
| `critical` | 强制安全模式 + 提供真实求助热线 + 现实求助建议 |

**内置热线**：
- 北京心理危机研究与干预中心 010-82951332
- 全国心理援助热线 400-161-9995
- 支持运营方通过环境变量覆盖

---

### 附加模块：主动关心（Proactive Care）

**不等用户开口，系统主动说第一句话。** 四种触发场景（按优先级）：

| 触发条件 | 关心文案 | 用意 |
|---|---|---|
| 连续 3+ 天情绪低分 | "兄弟，这几天情绪一直不太好。不用急着说什么，就是想问一句，还撑得住吗？" | 早期发现下坠螺旋 |
| 情绪曲线在好转 | "说实话兄弟，你这几天状态比之前好多了，自己感觉到了吗？继续，别回头。" | 强化正向变化 |
| 有计划但晚上没打卡 | "对了，今天的计划任务做了吗？做完跟我说一声，打个卡。" | 温和提醒不施压 |
| 超过 3 天没出现 | "好久没见，这几天怎么样了？" | 重新激活关系 |

**安全约束**：高风险时不触发（让安全流程接管）；同一天只触发一次

---

### 附加模块：运营管理后台（Admin）

**运营方可见**：
- 用户总量 / 日活 / 周活 / 月活
- 消息量趋势
- 对话模式分布（陪伴/分析/话术/恢复/安全占比）
- 情绪分布热力图
- 安全事件列表 + 实时告警
- 恢复计划统计（完成率/类型分布/打卡率）
- 单用户深潜（会话 → 消息 → 情绪趋势 → 关系实体）

**隐私保护**：只看匿名 ID，不看真实身份；看统计聚合，不看原文内容

## 三、用户旅程地图

### 旅程 A：初次使用（从痛到被接住）

```
打开网页 → 匿名登录（零注册摩擦）
    ↓
看到空状态引导卡片：
  💬 随时倾诉
  🔍 关系分析
  📅 恢复计划
  ✍️ 话术建议
    ↓
点击"随时倾诉" → 预填示例 → 修改为自己的话 → 发送
    ↓
看到"正在理解你说的话..."（thinking 状态）
    ↓
收到第一条回复：先共情 → 再洞察 → 最后建议
    ↓
感受：被理解了，而且回复很实在（不是鸡汤）
    ↓
继续聊 → 系统自动保持上下文 → 越聊越深入
```

### 旅程 B：反复来聊（从工具到习惯）

```
第二天打开 → 看到侧栏历史会话 → 点进昨天的对话继续聊
    ↓
系统记得昨天聊过的人和事件 → 不用重复背景
    ↓
第三天收到主动关心："好久没见，这几天怎么样了？"
    ↓
回复 → 系统注意到情绪趋势 → 可能建议"要不要试试恢复计划？"
    ↓
点击开始 7 天计划 → 每天收到任务 → 每晚打卡
    ↓
第 7 天打开成长页 → 看到情绪曲线在好转 → 自我认可
```

### 旅程 C：关系困惑（从混乱到清晰）

```
在对话中说"帮我分析一下他的行为"
    ↓
系统自动切到分析模式 → 从对话历史提取事实
    ↓
输出结构化卡片：
  📊 分析结论（基于 5 个心智模型）
  📋 证据列表（引用你说过的事实）
  ⚠️ 风险提示（2-4 条关系信号）
  💡 建议（一条可执行的下一步）
  📈 置信度 72%
    ↓
用户看完 → 说"那我应该怎么跟他说？"
    ↓
自动切到话术模式 → 输出 A/B/C 三个版本 → 选一个复制发送
```

### 旅程 D：危机介入（从失控到稳定）

```
用户发消息："我真的撑不下去了"
    ↓
关键词匹配 + AI 分类 → risk_level = high/critical
    ↓
所有分析功能自动锁死 → 切入安全模式
    ↓
回复：稳定化支持 + 呼吸引导 + 真实热线号码
    ↓
不分析、不评判、不建议"去运动" → 只做一件事：让用户觉得不是一个人
    ↓
后续消息继续在安全模式 → 直到风险降级
```

## 四、数据资产盘点

### 已有数据

| 数据类型 | 规模潜力 | 价值 |
|---|---|---|
| **情绪分类标签** | 每条用户消息一个（8 种情绪 × 9 种问题类型） | 情绪分布洞察、群体心理画像 |
| **关系模式库** | 自动抽取的 facts 列表 + 分析结论 | 典型关系模式识别（暧昧/冷暴力/PUA/...） |
| **对话上下文** | 完整对话历史（消息级别） | 训练数据、对话质量优化 |
| **情绪时序数据** | 每条消息的 emotion_state + risk_level + timestamp | 纵向趋势分析、早期预警模型 |
| **恢复行为数据** | 每日打卡：心情评分 + 反思文字 + 完成率 | 恢复路径效果验证 |
| **关系实体图谱** | 用户提到的人 + 关系类型 + 事件时间线 | 关系动态追踪 |
| **求助模式** | 用户使用功能的路径 + 频率 + 时段 | 产品优化依据 |
| **高风险语料** | 触发 high/critical 的消息（脱敏后） | 安全模型训练 |

### 可衍生洞察

- **"什么时候用户最脆弱？"**：peak_hours 数据已有（凌晨 1-3 点高频）
- **"哪种关系模式最常见？"**：issue_type 分布可直接统计
- **"7 天计划真的有效吗？"**：对比入计划前后的 emotion_score 均值
- **"话术教练被选最多的是哪个版本？"**：structured_json 可追踪选择
- **"用户平均聊几轮才进入分析模式？"**：session 消息数 + 模式切换点

## 五、竞争壁垒分析

### 已建立的壁垒

| 壁垒 | 说明 |
|---|---|
| **方法论 + 工程化** | 5 个心智模型 + 9 条决策启发式不只是提示词，而是写进了 orchestrator 路由逻辑、guard 检查、prompt 结构里。照抄提示词不等于照抄产品 |
| **安全流程** | 三层检测 + 四级响应 + 7 项输出守卫。这套流程是合规审查的基础，新竞品需要时间复刻 |
| **记忆连续性** | 跨会话记忆 + 自动实体抽取 + 摘要注入。用户用得越久，系统对其关系背景理解越深，迁移成本随时间上升 |
| **恢复计划数据** | 7 天/14 天打卡数据 + 情绪曲线是用户自己的成长记录，不舍得丢弃 |
| **结构化知识库** | 每次分析输出的 facts/evidence/risks/advice 不是一次性文本，而是结构化 JSON 可被后续查询、统计、优化 |

### 可被复制的部分

- 前端 UI / 颜色主题 / 布局
- 通用 prompting 技术
- SSE 流式聊天基础设施

### 不容易复制的部分

- 框架级方法论（需要情感领域专业知识 + 产品判断力）
- 安全流程 + guard 体系（需要法务 + 心理学 + 工程三方协作）
- 数据飞轮（用户数据越多 → 模型越准 → 体验越好 → 更多用户）

## 六、当前局限与短板

| 领域 | 局限 | 影响程度 |
|---|---|---|
| **用户获取** | 无内置增长机制（分享、邀请、社区） | ★★★ |

| **内容丰富度** | 恢复计划只有 2 种，场景覆盖有限 | ★★☆ |
| **多模态** | 仅文字，不支持语音/图片/表情包 | ★★☆ |
| **社交属性** | 纯单人工具，无社区、无互动 | ★★☆ |
| **专业背书** | 无心理咨询师认证 / 权威合作 | ★★☆ |
| **个性化深度** | 语气偏好只有 3 档，无法学习用户沟通风格 | ★☆☆ |
| **国际化** | 仅中文，无多语言支持 | ★☆☆ |

## 七、扩展方向（短期，1-3 个月）

### 7.1 更多恢复计划模板

**现有**：7 天失恋 / 14 天内耗

**可新增**：
- **21 天冷暴力脱离**：识别控制模式 → 建立边界 → 执行分离
- **7 天社交恢复**：分手后重建社交网络（不是找新对象，是找回朋友）
- **14 天自我价值重建**：针对被 PUA 后自我怀疑的用户
- **30 天独处训练**：面向恋爱依赖型用户，学会享受一个人

**技术成本**：只需新增 prompt 模板 + plan_type 枚举值，基础设施已完备

### 7.2 对话中的"复盘"功能

**场景**：用户发完消息后想回顾"我上次怎么处理的？"

**实现**：在对话中说"帮我回顾上次分析的结论" → 从 structured_json 检索历史分析卡片 → 重新展示

**价值**：强化记忆 → 验证建议是否被采纳 → 衡量建议效果

### 7.3 定时主动触达（Push 化）

**现有**：主动关心只在用户打开对话时触发

**升级**：
- 微信服务号 / 邮件 / 浏览器 Push
- 场景："你的 7 天计划第 3 天了，今天的任务是..."
- 场景："距离上次聊天已经 5 天了，想念你了"（用词需克制）

**风险**：push 过多 → 用户反感；需要精确的频率控制（每天最多 1 次）

### 7.4 情绪日记（独立于对话的记录功能）

**用户说**："有时候我不想聊天，就想记录一下今天的感受"

**实现**：
- 新增 `/journal` 页面
- 简单文本框 + 心情标签（高兴/平静/低落/焦虑/生气）
- 不调 AI，纯本地记录
- 但可选"要不要让系统帮你分析今天写的？"

**价值**：低门槛入口 → 不需要"有问题"才来 → 日常化使用 → 提高留存

## 八、扩展方向（中期，3-6 个月）

### 8.1 社区 / 匿名树洞

**设计**：用户选择把"脱敏后的分析结论"分享到公共时间线

**内容形态**：
- "暧昧 3 个月，系统分析显示投入不对等，置信度 72%"
- 其他用户可以"拍拍肩"（共情按钮）或"有同感"
- **绝对不显示原文**，只显示结构化摘要

**价值**：
- 用户发现"原来不只有我一个人有这个问题" → 去羞耻化
- 增长飞轮：社区内容吸引新用户 → 新用户使用工具 → 产出新内容

**风险**：需要严格内容审核（自动 + 人工），防止社区变成负能量放大器

### 8.2 关系双人视角（经对方同意）

**场景**：两人都在用这个工具，各自描述同一段关系

**实现**：
- A 邀请 B 建立"关系连接"（需双方确认）
- 系统可以交叉分析两人的描述差异
- 输出："你觉得对方冷暴力，对方觉得你施压。核心矛盾可能在于..."
- **双方各自的完整对话仍然完全隔离**

**价值**：从"单边叙事"升级到"双边理解"，大幅提升分析准确度

**风险**：隐私极其敏感，必须做到：双方都能随时退出、退出后数据完全分离

### 8.3 语音输入 + 情绪检测

**用户说**："打字好麻烦，我想直接说"

**实现**：
- 语音转文字（Whisper API）→ 输入到现有对话流
- 进阶：从语音特征检测情绪（语速、音量、停顿）→ 辅助 risk_level 判断

**价值**：降低输入门槛 → 适合情绪崩溃时连字都打不动的场景

### 8.4 专业咨询师桥接

**场景**：risk_level = critical 时，除了热线还可以一键预约在线咨询师

**实现**：
- 对接壹心理 / 简单心理等平台的 API
- 系统把脱敏后的情绪摘要传给咨询师（经用户同意）→ 咨询师不用从零了解
- 用户回来后系统知道"用户刚做完专业咨询"→ 调整陪伴策略

**价值**：产品从"自助工具"升级到"分级诊疗入口"

### 8.5 小程序迁移（抖音 / 微信）

**现有规划**：CLAUDE.md 从第一天就标注了抖音小程序迁移方向

**策略**：
- 后端 API + packages 完全不动
- 只替换前端层（React → 小程序框架）
- 匿名登录 → open_id 登录（字段已预留）
- SSE → 小程序环境的流式方案需要适配

## 九、扩展方向（长期，6-12 个月）

### 9.1 情感知识图谱

把用户群体的匿名化数据聚合，构建"中国年轻人情感问题知识图谱"：
- 典型关系模式库（暧昧拉扯型 / 冷暴力控制型 / 反复分合型 / ...）
- 每种模式的统计特征（持续时间、情绪曲线、最佳介入时机）
- 支撑分析模型从"通用框架"进化到"模式匹配 + 个性化建议"

### 9.2 AI 微调 / 蒸馏

用产品数据微调一个情感领域的轻量模型：
- 情绪分类更准（当前 emotion-intake 偶尔偏差）
- 安全检测更敏感（减少漏判）
- 回复更符合产品人设（减少 guard 拦截率）

### 9.3 多人关系支持

- 不只是"两人关系"，支持"家庭关系""室友关系""职场关系"
- 不同关系类型有不同分析框架
- 恢复计划也可以按关系类型定制

### 9.4 数据可视化报告

用户每月/每季度收到一份"情感健康报告"：
- 情绪波动曲线
- 最常聊到的话题
- 关系状态变化
- 恢复计划完成率
- "你比 3 个月前坚强了 XX%"（量化成长感）

## 十一、风险与红线

### 绝对不能做的事

| 红线 | 原因 |
|---|---|
| 把陪聊做成"虚拟恋人" | 制造依赖，与产品哲学矛盾，监管风险 |
| 泄露用户对话内容 | 法律 + 信任 + 产品根基 |
| 用危机数据做营销 | "10000 人想自杀但我们拯救了他们"——绝对不行 |
| 在 risk ≥ high 时推送广告/付费引导 | 用户脆弱时推销是剥削 |
| 用用户数据训练并出售模型（未脱敏） | 隐私法 + 伦理底线 |
| 承诺治疗效果 | 不是医疗器械，不能暗示"治愈抑郁" |

### 需要注意的灰色地带

| 场景 | 建议 |
|---|---|
| 用户频繁使用但不寻求线下帮助 | 定期建议"也可以和身边的人聊聊" |
| 未成年人使用 | 考虑年龄门槛或监护人知情功能 |
| 用户把系统当日记本（长期依赖但不成长） | 需要产品引导"下一步"而不是无限陪伴 |
| 单一用户消耗大量 AI 计算 | 合理限流，但不能让限流影响危机响应 |
